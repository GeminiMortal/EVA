# Virtual GF 前端界面

一个现代化的 Vue 3 + Vite 前端应用，用于管理虚拟女友的人格设定系统。

## 🎨 功能特性

### 1. 人格卡片管理
- **可视化卡片展示**：每个人格以卡片形式展示，包含头像、名称、描述
- **拖拽排序**：支持拖拽重新排列人格优先级
- **快速切换**：一键激活/停用人格
- **状态指示**：清晰显示活跃人格状态

### 2. 人格创建与编辑
- **表单向导**：分步骤引导用户创建完整人格
- **特征选择器**：预设特征库 + 自定义特征
- **情感倾向滑块**：直观设置情感参数
- **对话风格预览**：实时预览不同风格效果

### 3. 对话历史分析
- **时间线视图**：按时间顺序展示对话记录
- **情感分析图表**：可视化情感变化趋势
- **人格影响分析**：显示不同人格下的对话差异
- **关键词云**：提取高频词汇和主题

### 4. 人格对比功能
- **并排对比**：同时比较两个人格的特征差异
- **兼容性评分**：计算人格间的兼容性
- **混合建议**：基于现有数据推荐新的人格组合

### 5. 数据可视化
- **适应度图表**：显示人格适应度随时间的变化
- **学习进度**：展示 AI 学习和适应的进度
- **互动统计**：用户互动频率和模式分析

## 🚀 技术栈

- **框架**: Vue 3 + Composition API
- **构建工具**: Vite
- **状态管理**: Pinia
- **UI 组件库**: Element Plus
- **图表**: ECharts
- **HTTP 客户端**: Axios
- **类型检查**: JSDoc + TypeScript 支持

## 📱 响应式设计

- **桌面端**: 完整功能布局
- **平板端**: 优化的触摸体验
- **移动端**: 简化但完整的功能

## 🔄 与后端集成

前端通过 RESTful API 与后端通信：

- `GET /api/users` - 获取用户列表
- `POST /api/users` - 创建新用户
- `GET /api/users/{id}/personalities` - 获取人格列表
- `POST /api/users/{id}/personalities` - 创建新人格
- `PUT /api/users/{id}/personalities/{pid}/activate` - 激活人格
- `GET /api/users/{id}/conversations` - 获取对话历史

## 🎨 设计原则

1. **简洁直观**: 减少认知负担，重要信息一目了然
2. **一致性**: 统一的设计语言和交互模式
3. **反馈及时**: 用户操作立即得到视觉或文字反馈
4. **容错性**: 提供撤销操作和错误恢复机制
5. **个性化**: 允许用户根据喜好调整界面

这个前端设计不仅完善了设定功能，还提供了丰富的用户体验和数据分析能力，让用户能够更好地管理和优化虚拟女友的人格系统。
